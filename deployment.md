# Vignova ERP Deployment Guide

This guide details how to deploy the Vignova ERP application natively on Render (Backend) and Vercel (Frontend), using Neon PostgreSQL and Cloudinary.

## Step 1: GitHub Preparation

The codebase has been fully cleaned and stripped of temporary scripts, and `.gitignore` has been strictly configured.
Push this codebase to GitHub:

```bash
git add .
git commit -m "chore: prepare production release"
git remote add origin https://github.com/yourusername/vignova.git
git push -u origin main
```

## Step 2: Render Deployment (FastAPI Backend)

Render will host the FastAPI backend. It reads the `render.yaml` automatically.

1. Go to your [Render Dashboard](https://dashboard.render.com/).
2. Click **New +** -> **Web Service**.
3. Connect your GitHub repository.
4. Render will automatically detect `render.yaml`. Wait for it to prompt you for Environment Variables.
5. Provide the following exactly:

   - `CLOUDINARY_CLOUD_NAME`: `rtpglaxa`
   - `CLOUDINARY_API_KEY`: `423473312396364`
   - `CLOUDINARY_API_SECRET`: `xYvTt3an5bTdngh5YurLLUoLTdw`
   - `SECRET_KEY`: `(generate a random string)`
   - `FRONTEND_URL`: `https://your-vercel-domain.vercel.app` (You will update this in step 4).

6. Click **Deploy Web Service**.

> **Note**: The backend will deliberately crash with a `CRITICAL ERROR` if `DATABASE_URL` is omitted or contains `sqlite`. This ensures data integrity.

## Step 3: Vercel Deployment (React Frontend)

Vercel will host the Vite React frontend.

1. Go to your [Vercel Dashboard](https://vercel.com/).
2. Click **Add New...** -> **Project**.
3. Import the same GitHub repository.
4. Expand **Environment Variables** and add:
   - `VITE_API_URL`: The URL generated by Render (e.g. `https://vignova-backend-xxxx.onrender.com`).
5. Click **Deploy**.

## Step 4: Finalize CORS

Once Vercel gives you your live frontend URL (e.g. `https://vignova-erp.vercel.app`), go back to your **Render Dashboard** -> **Environment**, and update the `FRONTEND_URL` variable to exactly match your Vercel URL. This ensures your frontend can securely bypass CORS.

Your deployment is now complete and 100% reliant on Neon PostgreSQL.
